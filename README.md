# Geoglify

This project is meant to be a starting point for other web projects, especially those with geospatial needs, but it can be used for any kind of web application. The goal is to provide a modern, flexible structure that is ready to grow and adapt to different use cases.

## Tech Stack & Versions

Geoglify uses the latest versions of my favorite technologies:

- **[Laravel](https://github.com/laravel/laravel)** (PHP backend) - ^12.0
- **[PHP](https://github.com/php/php-src)** - ^8.4
- **[Vue.js](https://github.com/vuejs/core)** (JavaScript frontend) - ^3.5.25
- **[Vite](https://github.com/vitejs/vite)** (build tool) - ^7.2.4
- **[Vuetify](https://github.com/vuetifyjs/vuetify)** (UI framework) - ^3.7.0
- **[MapLibre GL](https://github.com/maplibre/maplibre-gl-js)** (maps library) - ^4.6.0
- **[PostgreSQL](https://github.com/postgres/postgres)** - 16
- **[PostGIS](https://github.com/postgis/postgis)** - 3.4
- **[Docker](https://github.com/docker)** and **[docker-compose](https://github.com/docker/compose)** (development environment)
- **[Tailwind CSS](https://github.com/tailwindlabs/tailwindcss)** - ^4.1.17
- **[Inertia.js](https://github.com/inertiajs/inertia)** - ^1.0.0

## Goals

- Be a starting point for web projects with geospatial features (maps, spatial data, etc.)
- Make it easy to add new tech and tools
- Promote good code organization and development practices
- Be easy to adapt for other types of web apps

## How to use

1. Clone the repository
2. Set up environment variables and dependencies
3. Start the services with Docker
4. Explore and adapt to your needs!
